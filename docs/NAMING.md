# Naming

> Related: [VISION.md](./VISION.md)

## Decision

| Field | Value |
|-------|-------|
| **Chosen name** | **Ticqex** |
| **Domain** | ticqex.com *(confirm registration)* |
| **Repo** | `ticqex` |
| **npm scope** | `@ticqex/*` |
| **API key prefix** | `tq_live_` |
| **Webhook signature header** | `X-Ticqex-Signature` |

## Etymology

Invented compound — reads as **tic** (ticket) + **q** (queue) + **ex** (exchange / API). Short, unique, no major conflicts found in support/ticketing OSS.

## Branding

| Asset | Value |
|-------|-------|
| **Tagline** | *API-first support, built for agents.* |
| **OSS product** | Ticqex |
| **Hosted (future)** | Ticqex Cloud |
| **Enterprise (future)** | Ticqex Enterprise (`/enterprise`) |

## What the name signals

| Signal | Fit |
|--------|-----|
| **Simple** | Short (6 chars), one word |
| **Agentic-first** | Neutral infrastructure tone — not "AI helpdesk" hype |
| **Developer-friendly** | Unique, ownable namespace; works as `@ticqex/core` |
| **Backend focus** | Doesn't imply customer portal |
| **Kanban / lanes** | Not explicit — carried by product, not name |
